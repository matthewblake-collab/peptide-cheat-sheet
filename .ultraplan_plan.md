# Implementation Plan: Peptide Cheat Sheet — hosted interactive reference

## Context
Greenfield static app (`index.html` + `data/peptides.json`) — searchable dark-theme card grid of ~90 Synex/LELE peptide products with per-product detail panels; data aggregated + attributed from peptide-db.com and peptidedosingprotocols.com; new public GitHub repo `matthewblake-collab/peptide-cheat-sheet`; Netlify auto-deploy on push to main; edit = push.

## Decisions (from interview)
- Repo: NEW `peptide-cheat-sheet` (public), local dir `~/Desktop/peptide-cheat-sheet`; `gh` authed as matthewblake-collab.
- Scope: ALL ~90 products in one pass. No-source items (Synex blends, solvents, Lemon Bottle…) = compiled-from-components, flagged `"compiled"` in `sources`.
- Netlify: fix CLI auth (browser login) → create+link site for GitHub auto-deploy + first `netlify deploy --prod`.
- Dark default + light toggle (`<html data-theme>`, persisted `localStorage.theme`). Fonts DM Mono / Syne (match existing index.html). ~13 use-case categories consolidated from peptide-db taxonomy.
- Mandatory: dismissable top disclaimer + persistent footer disclaimer (educational/research only, not medical advice, many compounds not human-approved, consult a physician); every record carries `sources[]`.

## Files to create
- `index.html` — full app: embedded CSS (dark/light vars), embedded JS; `fetch('data/peptides.json')` → render. Header (title, live search, category filter chips, theme toggle). Grid: cards grouped by category section headers (name, aka, tag, summary, risk dot) → click opens detail. Detail = slide-in panel (right desktop / full-screen mobile): Overview+mechanism · Dose & frequency · Cycle & on/off · Side effects & monitoring · Synergistic stacks (chips → open that product) · Comparison vs same-category alternatives · Sources (links). Close via ✕/ESC/backdrop. `#<id>` hash deep-links (shareable + powers synergy nav). Mobile-responsive `auto-fill minmax` grid.
- `data/peptides.json` — array of `{id,name,aka,category,summary,mechanism,dosing,cycle,sideEffects,synergies[],comparisons,sources[]}` for all ~90 products. `synergies` = other product ids.
- `data/products.map.json` — dev seed: deduped product list → {id,name,aka,category, peptidedb_slug?, pdp_slug?}.
- `scripts/fetch_sources.py` — curl each mapped slug (UA header, ~0.7s sleep, 1 retry), strip tags→text, write `scripts/raw/<id>.{peptidedb,pdp}.txt`.
- `netlify.toml` (publish `.`, no build, cache headers, 200 fallback), `.gitignore` (`.netlify/`, `.DS_Store`, `scripts/raw/`), `README.md` (how to edit + redeploy).

## Implementation Sequence
1. Scaffold: `mkdir ~/Desktop/peptide-cheat-sheet && git init`; write `.gitignore`, `netlify.toml`, `README.md`; build `data/products.map.json` (dedupe ~90, assign kebab ids, categories, source slugs).
2. Write & run `scripts/fetch_sources.py` → raw text dumps; log which slugs 404 / are JS-only.
3. Build `data/peptides.json`: fan out sub-agents in batches of ~12 products (prompt passes raw-dump file paths, not contents — keep <1500 tokens; on `prompt_too_long` fall back to direct authoring). No-dump items → compile from component records. Merge → validate `python -m json.tool`.
4. Write `index.html` (CSS + JS + disclaimers).
5. Test: `python3 -m http.server 8000` → smoke-test grid/search/filter/theme-persist/detail/synergy-nav/hash deep-link/mobile; zero console errors; JSON validates.
6. `git add -A && git commit`; `gh repo create peptide-cheat-sheet --public --source=. --remote=origin --push`.
7. Netlify: user runs `! netlify logout && netlify login` (browser) → `netlify init` (link GitHub repo, publish dir `.`, no build) → `netlify deploy --prod --dir .` → report live URL; `curl -sI <url>` = 200.

## Edge Cases & Risks
- `fetch()` fails on `file://` → must test via `http.server` (documented in README).
- Source pages occasionally JS-rendered → curl gets shell only; fallback = author from general knowledge, flag `"compiled"` (BPC-157 spot-check WAS server-rendered — mostly fine).
- Scrape volume ~150 reqs → UA header + sleep + 1 retry; skip+flag on persistent fail.
- Health/liability → disclaimers everywhere + source attribution; never invent dosing for source-backed items.
- Netlify CLI auth currently broken (stale dead site id elsewhere) → needs user browser login; isolated to step 7.
- Public repo → world-readable peptide reference (acceptable per user).
